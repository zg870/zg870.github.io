Gas优化

Gas是对全网节点资源消耗的付费凭证

节点需要为我们的交易消耗三类核心资源：

  • 计算资源（CPU）
  • 存储资源(storage)
  • 网络与验证成本

==其中storage是最贵的==

Gas优化 = 尽量少动 storage

  1. SSTORE（写storage)
    1. 最高可达20000gas
    2. 永久影响链状态
  2. SLOAD（读storage）
    1. 冷读 : ~ 2100 gas
    2. 热读 ：~100 gas
  3. KECCAK256
    1. mapping/array/struct slot 计算
    2. 使用频繁

Gas优化常用技巧

避免从0到1的内存写入

从零到非零的SSTORE是EVM中最昂贵的常规操作之一(首次占用一个新的 stroage slot)

SSTORE的gas取决于 storage slot的“原始值”与“新值”

热路径 = 合约中被调用频率最高、用户几乎必经的函数执行路径

  • 99%的用户交互
  • 都会经过这里
  • 每多1k gas，都会被无限放大

冷路径：只执行一次/极少执行/不影响用户交互的路径

热路径撞上 0 —> 非0

此处uint256 public activated;所有storage slot在合约部署后都是0，也就是说在部署完成的那一刻，activated == 0就已经存在了，但是只是逻辑上存在，并未发生过SSTORE写入

第一次调用transfer()的时候，是第一次访问activated.slot,并且SSTORE activated.slot = 1写入storage，EVM对比before : 0 , aftrer :1 触发0 —> 非0写入。

又因为transfer()是绝对热路径，所以第一次访问的人会多花费~20000gas

在与一些依赖精确gas估算的协议交互时，这种额外的20k gas可能导致交易因gas不足而失败，从而使该代币在实际使用中表现异常

正确设计

在部署完成时，所有的storage slot默认都是0，没有发生SSTORE操作

在构造函数中（冷路径）进行SSTORE操作，只需要执行一次，不属于任何用户交互且gas费由部署者承担。

冷热读问题

EIP-2929之后的SLOAD成本

冷读：第一次读取某个地址或者storage slot（~2100 gas）

热读：已读过，再次读取（~100 gas）

虽然第一次cold之后再进行读取都是warm，但是因为gas消耗是次数x成本,如果没有进行缓存，并将读取操作放入循环，那么花费的gas将会飙升。

变量packing

一个storage slot是32字节，256 bits，solidity按声明顺序填充slot，填不下才换下一个

虽然a，c两个小变量占用storage不到一个slot，但是，因为有一个32字节的变量b在他们之间，所以按照声明顺序，a b c必须各占一个slot

但是每多一个slot，成本就会上升

  • 每次读 = 多一次SLOAD
  • 每次写= 多一次SSTORE

建议在声明时将同类小类型的变量放在一起，bool/uint8/uint16靠前，uint256放在最后

只需要两个slot

struct packing

结构体中每个字段：独立分配slot，不会自动重排

与变量Packing一样，在声明时将同类小类型的变量放在一起，这使读取和写入结构体更便宜

storage pointer vs memory copy

solidity中所有复杂类型（struct/array/mapping)都必须带有数据位置，不是为了可读性，而是为了告诉编译器：数据存在于哪里，访问方式是什么，是否需要拷贝

memory（拷贝）

是一个完整的数据复制过程

真实语义是把users[id]整个struct从storage拷贝到memory

那对于结构体每一个字段，都会触发一次SLOAD,memory中每个字段都单独写入

==struct字段数 = SLOAD次数==

相应的Gas成本N 个字段 ≈ 2100 × N + 少量 MSTORE，当struct稍微复杂一点时，拷贝本身就是大头开销

那如果是动态类型的srtring/bytes

memory拷贝意味着：

  • SLOAD:读取name的storage slot
  • 计算string的实际存储位置(keccak256)
  • 循环SLOAD，每32字节写一次
  • MOSTORE：把所有内容写入memory
  • 更新 memory pointer

string越长，拷贝成本越高

storage pointer

零拷贝

本质：u不是一个数据，而是一个storage slot的地址描述

u.baseSlot = keccak256(id, users.slot)

等价于SLOAD(u.baseSlot + 0)

按需访问，只发生一次SLOAD

==对比之下，memory为了读一个字段，付出了整个struct的拷贝成本==

为了节省Gas，最好使用storage pointer而不是memory

利用Gas差异解决智能合约问题

对于同一address,第一次读取耗费的gas更高（冷读和热读）

我们可以利用Gas差异，让前后两次静态调用返回不同的值
