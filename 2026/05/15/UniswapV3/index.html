UniswapV3

一点点V3

UniswapV2资金利用率问题

在某一段价格区间内，池子中的流动性被交易实际使用的比例。它反映了价格变动过程中，有多少资金参与了交易

案例

假设ETH/DAI交易对的实时价格为1500DAI/ETH，交易对流动池中共有4500DAI和3ETH，根据$x\cdot y = k$可以算出池内的K值：$k = 4500\cdot 3 = 13500$

用x表示DAI，y表示ETH

初始阶段：$x_1 = 4500$，$y_1 = 3$

当价格下降到1300 DAI/ETH时有：

$x_2\cdot y_2 = 13500$

$\Large\frac{x_2}{y_2} = 1300$

得出$x_2 = 4192.54$，$y_2 = 3.22$

资金利用率：$\Large\frac{(x_1 - x_2)}{x_1} = 6.84%$

当价格上升到2200 DAI/ETH时有：

$x_2\cdot y_2 = 13500$

$\Large\frac{x_2}{y_2} = 2200$

得出$x_2 = 5449.77$，$y_2 = 2.48$

资金利用率：$\Large\frac{(x_1 - x_2)}{x_1} = 21.11%$

==价格波动，资金利用率很难高于25%==

UniswapV3是为了解决资金利用率过低的问题

UniswapV3可以减少LP损失

如果ETH初始价格是 2000DAI

LP设置的价格区间是[1500,2500]

当ETH价格降低到0的时候，因为价格区间限制，ETH到达1000DAI就不再降低。那也就是说，不论外界价格如何，LP最低拥有1000DAI；

如果在UniswapV2中，LP最低拥有的DAI能降低到0，从而亏损较大

集中流动性/虚拟流动性

在UniswapV3中，LP不再需要在整个价格区间提供流动性，而是可以指定一个价格区间

$[P_\min, P_\max]$,来集中提供流动性

  • 流动性只在$[P_\min, P_\max]$范围内有效
  • 当价格超出该范围，LP的一侧资产全部转化为单一资产
  • 资金只有在市场价格进入该区间时才参与交易，赚取手续费
  • 资金利用率极大提高

价格区间$[P_a, P_b]$

$\Large x\cdot y = k = L^2$

$\Large P = \frac {y}{x}$

$\Large x = \frac {L} {\sqrt{P}}$

$\Large y = L\sqrt{P}$

$\Large P = P_b$时

$\Large x = x_v + x_r = \frac {L}{\sqrt{P_b}}$

因$x_r = 0$得出 $x_v = \frac{L}{\sqrt{P_b}}$

$\Large P = P_a$时

$\Large y = y_r + y_v = L\sqrt{P_a}$

因$y_r = 0$得出$y_v = L\sqrt{P_a}$

$\Large x\cdot y = L^2$ —-> $\Large (x_r+x_v)\cdot(y_r + y_v) = L^2$

限价订单

V3 中的 价格和tick

价格也是由tick表示的

  • 价格离散为tick（整数索引）

  • 每一个tick对应一个价格

    $P = 1.0001^{tick}$

  • 流动性和价格变化以tick为单位计算

例子

tick = 0 P = 1

tick = 100 P ~ 1.01005 (上涨约1%)

tick = -100 P ~ 0.99005 (下跌约1%)

  • 将价格空间分割成离散格子
  • 每个LP可以选择再特定tick区间提供流动性
  • 计算swap或LP资产时方便定位价格区间

Tick与流动性的关系

在价格区间$\Large [P_{lower}, P_{upper}]$,流动性为L价格P在区间内

高价代币y的数量

$\Large y = L\cdot (\sqrt{P} - \sqrt{P_{lower}})$

低价代币x的数量

$\Large x = L\cdot (\frac{1}{\sqrt{P}} - \frac{1}{\sqrt{P_{upper}}})$

在区间$[P_{lower}, P_{upper}]$内，L保持不变，x,y的比例随价格变化

当价格到达边界，tick流动性耗尽，系统自动切换到下一个tick区间
