Lending

Lender(出借人):存入资金的人，获得利息收益

Borrower(借款人):借入资金的人，需要支付利息，并提供抵押

Protocol(协议):只能合约自动执行借贷逻辑

流程

==存款==

  1. 用户将加密资产存入协议
  2. 协议发放对应代表代币表示用户的存款份额和利息权利
  3. 利息随时间积累，份额不变，但可兑换的代币增加

==借款==

  1. 借款人先抵押一定数量的资产

  2. 协议根据抵押率(Collateral Factor)决定可借额度

    1 ETH（价值2000 USDC），抵押率75%，可借1500USDC

  3. 借款人支付利息，利息随市场供需浮动

核心机制

==超额抵押==

DeFi借贷大多要求借款人抵押比借款更高的资产，防止违约

抵押率一般为50% - 80%

==利率模型==

  1. 算法利率，利率随供需变化

    1. 借款需求高 -> 利率上升 -> 吸引更多存款
    2. 借款需求低 -> 利率下降 -> 降低借款成本

  2. 常见公式

    $\Large r_{borrow} = r_{base} + k\cdot utilization$

    $\Large utilization = \frac {已借出} {总借款}$

==清算==(Liquidation)

  • 如果借款人抵押率下降到安全线以下，协议会部分或全部清算借款
  • 清算者(Liquidator)可以折扣价购买抵押资产，借款人承担损失

==$\Large 安全线 = \frac{抵押资产价值}{借款金额} \cdot 100%$==

GMX

GMX是去中心化杠杆交易平台（去中心化的永续合约+ 现货交易平台）

GMX支持杠杆交易和做空，用户可以在链上直接进行高达50x的杠杆交易

  • 主网：Arbitrum&Avalanche
  • 类型：去中心化永续交易所
  • 特点：无资金费率操控、无滑点（使用链上预言机定价）

核心机制

  • Trader（交易者）：做多/做空交易
  • Liquidity Provider（GLP）：提供资金池流动性，赚手续费
  • Oracle（预言机）：提供价格

==做多/做空交易==

  1. 做多交易(Long)
    1. ETH当前的价格2000USDT
    2. 预期ETH会涨
    3. 买1ETH，用2000USDT
    4. 价格涨到2500USDT，卖出
    5. 盈利500USDT
  2. 做空交易（Short）
    1. ETH当前的价格是2000USDT
    2. 认为ETH会跌
    3. 从市场借1ETH然后卖出，获得2000USDT
    4. ETH降价到1500USDT
    5. 用1500USDT买回1ETH还掉
    6. 盈利500USDT

GLP机制

GLP是GMX的流动性代币，代表一个多资产资金池

这个池子为所有交易者提供对手方流动性

  • GLP由多种资产组成（例如ETH、BTC、USDT、DAI等）
  • 用户随意存入任意资产即可获得GLP
  • GLP价格代表整个池子的净资产价值

==对手流动性==

想买（做多）或卖（做空）某个资产时，必须有人在另一边与你交易

你拿1000USDT作为保证金，系统需要借出一部分资金给你买ETH，这笔借出的ETH来自流动性池。对手方并不是另一个交易者，而是GLP持有人；GLP池子承担价格风险、赚取手续费和资金费率。

GLP的收益来源

  • 交易手续费（开仓、平仓、换币）
  • 清算收益（交易者被清算）
  • 借贷利息（杠杆头寸的借款费）

==头寸==：头寸就是你当前所持有的仓位，包括方向（多/空）、数量、入场价格、杠杆、未实现盈亏等。多头(Long Position)，押注价格上涨，持有ETH，看涨；空头(Short Position)，押注价格下跌，借ETH卖出，等跌后买回。

例：

​ 以2000USDT开多1ETH，杠杆5x，当前价格2200

​ 头寸价值 = 2200 USDT

​ 盈利 = (2200 - 2000) x 1 = 200 USDT

头寸 = 市场方向 + 持仓规模 + 开仓价格 + 未实现盈亏

==开仓==

开仓 = 建立一个新的头寸

  1. 存入保证金（Colletral)
  2. 指定方向（做多/做空）
  3. 系统按杠杆倍数放大仓位
  4. 成交后，记录为一个活跃头寸

例：

用500USDT做5x多头：

​ 实际仓位规模：500 x 5 = 2500 USDT

​ 系统帮你买入2500USDT价值的ETH

​ 你此时持有**[多头头寸]**

开仓 = 存钱 + 借钱 + 买/卖资产 + 记录仓位

==平仓==

平仓 = 结束你的头寸，把借入的资产还回去，结算盈亏

例：

以前以2000USDT买入1ETH（做多）

​ 当ETH涨到2200USDT –> 你卖出ETH

​ 获利200USDT

​ 剩下的钱（本金 + 利润）回到你的账户

如果跌到1800USDT –> 亏损200USDT –> 平仓后账户减少

==换币（Swap/交换）==

换币就是把一种代币兑换成另一种，比如USDT –> ETH。本质是一次无杠杆的即刻交易（spot trade)

用1000USDT直接换0.5ETH，这叫Swap，没有借贷，没有杠杆，不会被清算

用1000USDT做多ETH，用杠杆买更多ETH，这叫开多头仓位，有借贷，有杠杆，可能被清算

平仓 = 反向交易 + 归还借款 + 结算盈亏

GLP的风险

  • 交易者亏钱–>GLP收益
  • 交易者赚钱–>GLP亏损

==GLP的收益和交易者表现负相关==

GMX交易流程

以做多ETH为例

  1. 交易者存入抵押物（例如1000USDC）
  2. GMX根据抵押额和杠杆倍数创建一个头寸
  3. 平台从GLP池借出资金（4000USDC）补足杠杆
  4. ETH价格上涨–> 交易者盈利
  5. ETH下跌 –> 交易者亏损，如果跌幅超过保证金 –> 清算

==定价机制（无滑点）==

  • 使用Chainlink预言机价格，而不是AMM价格曲线
  • 这样交易者不会因滑点损失，流动性提供者承担真实盈亏

GMX代币双系统

GMX主要有两个代币

GMX是平台的治理代币，同时也是“协议利润的分红凭证”，持有者不直接参与交易，而是分享平台交易产生的收入

主要的收入来源是：

  • 交易手续费（Swap + 开仓 + 平仓）
  • 借贷利息（杠杆交易借出的资金利息）
  • 资金费率

GLP是GMX的流动性池代币，代表在池子中的份额。

本质：当交易者做多或做空时，对手方就是GLP池，GLP池赚手续费但同时承担价格波动风险。

主要收入来源是：

  • 交易手续费
  • 借贷利息
  • 清算收益（交易者爆仓时）
  • 资金费率

收益分配机制：

收入来源 分配对象 比例
交易/借贷手续费 GMX持有人&GLP持有人 各占50%
收益形式 ETH/AVAX（原生代币） 直接分红

GMX是一个跨链协议，目前主要部署在：

  • Arbitrum链（主链），收益以ETH结算
  • Avalanche链，收益以AVAX结算

质押GMX或持有GLP不会拿到额外代币奖励，而是拿到链上真实赚到的原生代币（ETH或AVAX）

每笔交易（Swap / 开仓 / 平仓 / 借贷 / 清算）都会产生手续费：

  • 比如用户在 Arbitrum 链上交易 → 收的手续费是 ETH；
  • 系统定期把这些 ETH 累积到一个分红池；
  • 然后按照比例（50%/50%）分配给 GMX 质押者和 GLP 持有人。

==直接分红==：质押GMX或GLP后，系统会自动周期性地把协议产生地ETH/AVAX分红到你的地址，无需手动领取代币或参与通胀活动。

清算机制

GMX使用保证金比率控制风险：

  • 清算金比例 < 阈值 （一般 10% - 13%）—> 清算触发
  • 清算者会偿还债务，获得清算奖励
  • 多余抵押返给用户

GMX收费模式

开仓费：0.1%

平仓费：0.1%

借款费：按小时计算

兑换费：Swap时的手续费（0.2%~0.8%)

Aave

Aave的核心是一个去中心化的借贷协议（去中心化资金池），

  • 用户可以存入资产赚取利息（作为出借方）；
  • 抵押资产借出其他代币（作为借款方）

==完全通过智能合约运作，无需银行或中介==

核心机制

资金池

每个支持的资产（如ETH，USDT、DAI、WBTC）都有独立的资金池

存入者将资产打入池子，借款者从池子中借走

Aave是池化借贷，不需要匹配出借人和借贷人

aToken收益凭证

在Aave存入资产时，会获得对应的aToken

  • 存入ETH —> 得到aETH
  • 存入USDC —> 得到aUSDC

存入资产后，aToken（存款证明&利息增长凭证）的余额会自动增长，反映存款利息

借款与抵押

借款需要抵押足额资产

  • 抵押1ETH （价值2000USDT）
  • 最多可以借出约1500USDT

如果ETH价格下跌，健康因子下降，一旦跌破清算底线，就会被清算人执行操作，卖出抵押品偿还债务

健康因子：衡量借款人的账户安全程度，也就是你的抵押资产能否安全覆盖当前债务的比率

它由Aave合约实时计算

$\Large Health Factor = \frac{抵押资产价清算阈值}{借款资产价值}$
